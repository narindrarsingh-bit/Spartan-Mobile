# Spartan Native — Handoff

## Current Goal
Build a polished Capacitor native wrapper for the Spartan CLI web terminal. UI should look like a legitimate Discord-style productivity app (navy/gold), not a raw terminal.

## Architecture
- **Wrapper repo**: `~/Documents/spartan-native` (Capacitor)
- **Source web app**: `~/Documents/spartan-cli/dist/` (read-only, never modify)
- **Capacitor config**: `capacitor.config.json` (appId: com.roy.spartancli)
- **Web files**: `public/` (index.html, styles.css, app.js, icon.png)
- **Native platforms**: `android/`, `ios/` (generated by `npx cap sync`)
- **Spartan backend**: `http://100.78.120.128:8797` (Tailscale)
- **WebSocket endpoint**: `/terminal?session=native&profile=<profile>&token=...`
- **Message format**: JSON — `{type: "ready"|"output"|"error"|"input"}`

## Failures & Blockers
1. **Java version mismatch**: Fedora has Java 25/26, Gradle 8.14.3 requires Java 17 or 21. Cannot build Android APK without `sudo` to install older Java.
2. **No macOS environment**: Cannot build iOS IPA. GitHub Actions CI runner is an option.
3. **iPhone preview failed** (grey screen):
   - Root cause #1: CSS `:root` variables were overwritten/lost during file writing (the initial append was replaced by a subsequent write). This was fixed in this session.
   - Root cause #2: `app.js` references `profileSelect = document.getElementById("profile-select")` but `index.html` has no such element — will cause a JS runtime error on init.
   - Root cause #3: CSS file is still incomplete — missing input area, settings panel, and toast CSS.
   - Root cause #4: iOS Safari cache may still serve old assets. Needs hard refresh.
4. **CSS file writing problem**: `write_file` overwrites entirely. Need to write complete file each time, not chunk-and-append.

## Current Blocker
The frontend is **partially broken**:
